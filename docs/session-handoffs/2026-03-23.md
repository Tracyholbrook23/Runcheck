## What Changed

One feature set landed after the 2026-03-22 handoff was generated (late evening, 2026-03-22):

**Username system** — Every RunCheck account now requires a unique `username`. Chosen at signup by new users and claimed via a migration screen by existing accounts.

### Key changes

**SignupScreen.js** — Form extended with a `username` field (plus separate `firstName` / `lastName` fields in place of the single `name` input). Client-side validation: `USERNAME_REGEX`, `EMAIL_REGEX`, `DOMAIN_TYPOS` typo detection, and live password-requirement checklist. Firestore profile write moved out of this screen entirely — SignupScreen now only creates the Firebase Auth account and sends the verification email, then passes `signupData` as route params to VerifyEmailScreen.

**VerifyEmailScreen.js** — On email confirmation for a new signup: atomically writes `users/{uid}` profile + reserves `usernames/{usernameLower}` in a single Firestore transaction. Idempotent: skips reservation if the username doc already belongs to this uid. For returning users without a `username` field, routes to `ClaimUsername`.

**ClaimUsernameScreen.js (NEW)** — Migration gate shown to authenticated users whose `users/{uid}` doc is missing the `username` field (accounts created before the username system). Claims username via `USERNAME_REGEX` validation + Firestore transaction (reserve `usernames/{usernameLower}` + set-merge `users/{uid}`). Routes to Main if `onboardingCompleted`, else OnboardingWelcome.

### Schema additions
- New `usernames/{usernameLower}` collection — `{ uid, createdAt }`. One doc per reserved username.
- `users/{uid}` — added `username`, `usernameLower`, `firstName`, `lastName`, `phoneNumber: null`.

- All features from the 2026-03-22 handoff remain working (Run Level, Run Chat expiry + unread, EditProfileScreen, DM system, Push Notifications Phase 1, etc.)

## Bugs / Risks

- **`usernames` Firestore rules missing** — Until rules are written, any authenticated user can overwrite another user's username reservation. Low immediate risk (requires knowing another user's usernameLower), but a security gap.
- **DM security gap remains** — Do not invite external users to test DMs until `conversations` Firestore rules are deployed.
- **Run Chat permission-denied on device** — blocks device QA of Run Chat until rules are deployed.
- **SignupScreen comment mismatch** — The JSDoc at the top of `SignupScreen.js` still describes the old flow (Firestore write in signup). Code is correct; comment is stale.
- **Username edge case on retry** — If SignupScreen Auth creation succeeds but the user closes the app before reaching VerifyEmailScreen, they'll have an Auth account but no Firestore profile. On next launch, they'll hit the ClaimUsername gate if they log in. `set({ merge: true })` in ClaimUsername handles this gracefully. But they'd need to re-enter their signup data manually (name, age, skill level) via EditProfileScreen.
- **OTA updates broken on current binary** — frontend bug fixes cannot ship to existing TestFlight users until fresh build.
