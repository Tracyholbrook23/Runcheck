- **`services/dmService.js`** (new) — Complete DM service: `openOrCreateConversation` (idempotent, deterministic conversationId), `subscribeToConversations` (real-time inbox), `sendDMMessage` (message + conversation metadata), `markConversationSeen` (unread clearing via dot-notation field path).

## Bugs / Risks

- **No DM Firestore rules** — this is a security gap. Do not publicize the Messages feature until rules are deployed.
- **`sendDMMessage` is not transactional** — `lastMessage` on the conversation doc could lag by one message under a tight race condition. Acceptable at MVP scale but worth noting.
- **`useMyRunChats` does N `getDoc` calls** (one per active run) — fine for 0–3 runs per user but will cause read spikes for power users. Acceptable for MVP.
- **`notifCooldowns` map growing unboundedly** (known issue from 2026-03-20, still unresolved) — migrate to subcollection before ~500 active users.
