## What Changed

Four features landed today across 10 files:

**Run Level Phase 1** — Runs now carry a `runLevel` tag (`casual | mixed | competitive`) set by the creator at creation. Picker added to the Start-a-Run modal in RunDetailsScreen. Color-coded badge on run cards (Mixed is hidden as neutral default). Filter added to the ViewRunsScreen filter bottom sheet. Backwards compatible with old runs (absent field treated as `'mixed'`). Client-side only — no backend deploy.

**Run Chat expiry** — Each run's chat window closes 4 hours after `startTime`. `chatExpiresAt` is now written by `startOrJoinRun`. `RunChatScreen` shows a read-only "This run chat has ended" state (input bar replaced). `useMyRunChats` hides expired chats from the Messages inbox. Firestore `isChatActive()` rule hard-blocks new messages after expiry.

**Run Chat unread detection** — `sendRunMessage` now stamps `lastMessageAt` on the run doc (fire-and-forget). `markRunChatSeen` writes `lastReadAt` on the `runParticipants` doc when the user opens the chat. `useMyRunChats` computes `isUnread = lastMessageAt > lastReadAt` and exposes `runChatUnreadCount`. HomeScreen Messages badge updated to total `dmUnreadCount + runChatUnreadCount`.

**EditProfileScreen** — New screen for editing Display Name and Skill Level. Writes to Firestore `users/{uid}.name` and Firebase Auth `displayName`. Reached via Settings → Account Info in ProfileStack.

## Bugs / Risks

- **DM security gap is the most urgent risk.** Do not publicize or invite external users to test DMs until `conversations` Firestore rules are deployed.
- **Run Chat permission-denied on device** — blocks device QA of Run Chat until rules are deployed.
- **OTA updates broken on current binary** — frontend-only bug fixes cannot ship to existing TestFlight users until the next build.
- **`lastMessageAt` / `lastReadAt` absent on old runs** — `useMyRunChats` treats absent fields conservatively as read (no phantom badges). Correct behavior but means old run chats always appear as read until the first new message after this deploy.
- **`notifCooldowns` map growing unboundedly** — migrate to subcollection before ~500 active users (logged in BACKEND_MEMORY Known Issue #9).
- **`sendDMMessage` is not transactional** — `lastMessage` on the conversation doc can lag by one message under a tight race condition. Acceptable at MVP scale.
- **`useMyRunChats` does N `getDoc` calls** (one per active run) — fine for 0–3 runs but causes read spikes for power users. Acceptable for MVP.
