### Reliability Formula Repair (2026-03-25)

**`scripts/repairReliabilityScores.js`** — NEW one-time admin repair script. Scans all `users` docs, recalculates `reliability.score` from stored counters (`totalAttended`, `totalNoShow`, `totalLateCancelled`) using the canonical formula: `totalAttended < 3 → 100`; else `clamp(100 − 20·noShows − 8·lateCancels, 0, 100)`. Dry-run by default; `DRY_RUN=false` to commit. Fixes users whose scores were set by older backend code before the `totalAttended < 3` lock existed.

### Centralized Input Sanitizer (2026-03-25)

**`utils/sanitize.js`** — NEW. Seven pure, silent, keystroke-safe sanitizer functions:
- `sanitizeUsername` — [a-z0-9._], max 20, lowercases
- `sanitizeName` — printable name chars (accented letters, digits, space, ' . -), max 40
- `sanitizePersonName` — same as sanitizeName but max 30
- `sanitizeSearch` — [a-zA-Z0-9 '.-&], max 50
- `sanitizeFreeText` — strip control chars only (preserves \t \n \r), default max 500
- `sanitizeAddress` — printable address chars (letters, digits, . , - ' # /), max 200
- `sanitizeState` — [A-Z], max 2, uppercases

**Root cause fixed:** `onSnapshot` callbacks were firing while React Navigation held the JS thread for navigation animations. The result was that state updates queued but couldn't render until the user touched the screen ("frozen skeleton" symptom). Deferring via `InteractionManager` lets the animation fully complete before any subscription opens.
