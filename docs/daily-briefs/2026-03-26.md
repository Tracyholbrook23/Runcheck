### 2. Write + deploy Firestore rules for `conversations` and `usernames`

Add both rule blocks in the same file, then deploy once.

**`conversations` pattern:**
- `conversations/{conversationId}`: read/write only if `request.auth.uid in resource.data.participantIds`
- `conversations/{conversationId}/messages/{messageId}`: create allowed if participant + `isNotSuspended()`; update/delete denied
- Message reads: participant only

**`usernames` pattern:**
- Write only if `request.auth.uid == request.resource.data.uid`
- No client reads needed (uniqueness check happens via transaction)

Deploy with the same `firebase deploy --only firestore:rules` command.

**File:** `runcheck-backend/firestore.rules`
