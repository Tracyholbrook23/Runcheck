### 2. Write and deploy Firestore rules for `usernames` collection
**Why:** Security gap. Any authenticated user can overwrite another user's username reservation right now.
**File:** `runcheck-backend/firestore.rules`
**What to add:** Write rule on `usernames/{usernameLower}` — only the owner `uid == request.auth.uid` can create; no updates or deletes allowed. No reads needed by other users.
**Then deploy:** `firebase deploy --only firestore:rules`

---

### 3. Write and deploy Firestore rules for `conversations` collection
**Why:** DMs are a complete security gap — any authenticated user can read or write any conversation.
**File:** `runcheck-backend/firestore.rules`
**What to add:** Read/write on `conversations/{conversationId}` and its `messages` subcollection should require `request.auth.uid` to be one of the two participants (derivable from the deterministic `conversationId = [uid_a, uid_b].sort().join('_')`).
**Then deploy:** `firebase deploy --only firestore:rules`
