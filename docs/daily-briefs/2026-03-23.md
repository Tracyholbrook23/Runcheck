## TOP 3 Priorities Today

### 1. Write + Deploy Firestore rules for `usernames/` collection
**Why:** Username reservation is live in production with zero rules. This is an open write vulnerability.
**Start here:** `firestore.rules` — add rules for `usernames/{usernameLower}` (allow read: any authenticated user; allow write: only if `request.auth.uid` matches the `uid` field in the document, and only if the document doesn't already exist).

### 2. Write + Deploy Firestore rules for `conversations/` collection
**Why:** DMs are stored in Firestore with no access control. Any authenticated user can read any conversation.
**Start here:** Same `firestore.rules` file — scope reads/writes to participants only (`request.auth.uid in resource.data.participantIds`).

### 3. Deploy Run Chat Firestore rules
**Why:** Rules are already written — this is just a deploy step. Low effort, closes a known gap.
**Start here:** Confirm rules exist in `firestore.rules`, then `firebase deploy --only firestore:rules`.
